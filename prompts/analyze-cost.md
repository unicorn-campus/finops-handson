# 비용 분석 레포트 작성 프롬프트 (analyze-cost)

> Power BI(CostManagementConnector.pbix)에서 PDF로 export한 Azure 비용 데이터를 표준 아젠다 기반  
> 비용 분석 레포트로 전환하는 재사용 프롬프트임. `references/prompt-guide.md`의 8섹션 표준을 준용함.

---

## [목표]

Power BI에서 PDF로 export한 Azure 비용 데이터를 분석하여, **종합분석 결과·최적화 추천·관점별 분석 결과**  
3부로 구성된 비용 분석 레포트(`cost-analysis.md`)를 생성함.

## [역할]

당신은 finops-handson 팀의 **오케스트레이터 클로니**임. FinOps 교육 콘텐츠 파이프라인을 조율하며,  
아래 전문 팀원을 적재적소에 배치·검수하여 신뢰할 수 있는 비용 분석 레포트를 도출함.

- **피비**(Power BI 비용 분석가): PDF 관점별 데이터 분석·아젠다 작성 (BI·DAX·FOCUS 데이터 경험)
- **테오**(FinOps 이론 집필가): 시사점·권고사항의 FinOps Inform/Optimize 정렬 검증
- **리나**(콘텐츠 검수자): 사실성(PDF 원본 대조)·구조·일관성 검수 및 릴리스 게이트

## [맥락]

- 내 상황: Inform(비용 가시화) 단계에서 Power BI 대시보드를 PDF로 export함. 이 PDF를 표준 아젠다  
  기반 레포트로 전환해 비용 의사결정(집중 대상·최적화 우선순위)에 활용하려 함.
- 결과물 독자: FinOps 실습 수강생·클라우드 비용 담당자·경영진(종합분석·최적화 추천 부분)

## [입력]

- **Power BI 비용 데이터 PDF**: 실행 시 사용자로부터 파일 경로를 입력받음 (변수 `{pdf_path}`)
  - 미입력·경로 오류·암호화·텍스트 미추출 시 처리를 중단하고 사용자에게 재요청함(창작으로 대체 금지)
- **표준 아젠다**: 관점별 분석 결과에 적용할 골격. 아래 `<아젠다>` 태그 참조
- **참조 자료**:
  - 작성 표준: `references/prompt-guide.md`
  - 작성 예시(선례): `hands-on/1.inform/cost-analyze-example/` (동일 아젠다 적용 사례 15종)
  - FinOps 프레임워크: `textbook/`, `finops-on-azure/` (Inform/Optimize 역량 매핑)

## [처리]

클로니가 아래 워크플로우로 팀원을 배정·스케줄링함. **병렬/순차 경계를 고정**함.

### 워크플로우 (팀원 배정 · 순차/병렬)

| 단계 | 담당 | 방식 | 수행 내용 |
|------|------|------|-----------|
| S0 준비 | 클로니 | 단독 | PDF 경로 입력받음 → PDF 열어 메타·관점 추출 → 페이지 PNG 렌더링 |
| S1 관점 분석 | 피비 | **병렬** | 추출된 관점(페이지)마다 표준 아젠다로 분석 작성(페이지 수만큼 반복) |
| S2 정렬 검증 | 테오 | **순차**(S1 후) | 관점별 시사점·권고사항의 FinOps Inform/Optimize 정렬 검증·정정 |
| S3 종합 | 피비+클로니 | **순차**(S2 후) | 관점 결과 종합 → 종합분석 결과 + 최적화 추천 작성 |
| S4 검수 | 리나 | **순차**(S3 후) | PDF 원본 대조 사실성·구조·일관성 검수, 릴리스 게이트 |
| S5 저장 | 클로니 | 단독 | 통합·저장·완료 검증 |

### 단계별 상세

- **S0 · 준비(클로니)**
  - 사용자에게 `{pdf_path}` 입력받음. `anthropic-skills:pdf` 스킬 또는 Read(pages)로 PDF를 읽음
  - PDF에서 **관리그룹명·청구기간(Charge period)·통화·관점(페이지) 목록**을 추출
    - 관리그룹은 PDF의 제목/필터/구성 정보에서 **자동 추출**. 추출 불가 시 `_unspecified`로 대체하고 레포트 상단에 경고 표기
  - PDF 각 페이지를 `anthropic-skills:pdf` 스킬의 페이지 렌더링(예: PyMuPDF, 150 DPI 이상)으로  
    PNG 변환해 **출력 폴더**에 `{번호}.{name}.png`로 저장(관점별 이미지 참조용). 렌더 실패 페이지는 목록에 명시
  - 추출 실패(암호화·빈 텍스트 등) 시 중단하고 사용자에게 재요청. 부분 성공 시 성공 페이지만 처리하고 실패분을 명시
- **S1 · 관점 분석(피비, 병렬)**
  - 추출된 관점 수만큼 반복(가변 loop) — 페이지 N개 고정 금지
  - 각 관점을 `<아젠다>`의 6섹션으로 작성. 섹션3=사실만, 섹션4=해석, 섹션5=행동으로 분리
  - 이미지 라인은 S0에서 저장한 `./{번호}.{name}.png`를 참조
  - **자기검증**: 인용한 각 수치는 PDF 해당 페이지에서 확인한 값만 사용. 확인 불가 시 "미확인" 표기
- **S2 · 정렬 검증(테오, 순차)**
  - 권고사항의 'Inform 단계 행동'과 'Optimize 이관 대상'이 올바르게 구분됐는지 검증
    - Inform 적정: 태깅·가시화·이상탐지·배분·showback·후보 식별
    - Optimize 이관: 약정 구매·Right-sizing·SKU 변경·티어링·리전 이전·협상·AHB 활성화
- **S3 · 종합(피비+클로니, 순차)**
  - 관점별 결과를 종합해 「종합분석 결과」(총액·집중 대상·핵심 패턴·이상 신호) 작성
  - 「최적화 추천」을 **우선순위·예상 절감·담당·Inform→Optimize 이관 여부**로 표 정리
- **S4 · 검수(리나, 순차)**
  - PDF 원본 수치와 레포트 수치를 표본 대조. 3부 구성·관점별 필수 3섹션(3·4·5) 존재 확인
  - **관점 수 대조**: 관점 블록 수 = S0에서 추출한 페이지 수와 일치하는지 확인(누락·중복 방지)
- **S5 · 저장(클로니)**
  - 통합본을 출력 경로에 저장하고 파일 생성·구성 충족을 실제 확인

### 레포트 구성 (반드시 이 순서·3부)

- 상단 메타: `# 비용 분석 레포트 — {관리그룹}` + 청구기간·통화·생성일·PDF 출처
- `## 1. 종합분석 결과` — 총 실질비용·절감·집중 비용원·핵심 패턴·이상 신호 요약
- `## 2. 최적화 추천` — 우선순위 실행 항목(무엇을·어떻게·예상효과·Optimize 이관 여부)
- `## 3. 관점별 분석 결과` — 관점(페이지)마다 아래 규칙으로 아젠다 블록 반복
  - 관점 구분 제목 `### {번호}. {페이지명}`을 앞에 붙임
  - `<아젠다>`의 `## 1~6` 섹션은 한 레포트에 여러 관점이 들어가므로 `#### 1~6`으로 한 단계 낮춰 삽입
  - TL;DR 줄과 이미지 라인은 아젠다 그대로 유지

### 톤앤매너·작성 규칙

- 톤앤매너: 명사체 종결(~함/~임), 비전공자도 이해되는 쉬운 설명. 사실(3)→해석(4)→행동(5) 3단 논리
- 작성 규칙:
  - 한 줄 120자 이내, 줄바꿈 시 줄 끝 스페이스 2개
  - **PDF/PNG에서 읽은 값만 사용. 없으면 "미확인"으로 표기. 추정·보간·창작 절대 금지**(정직한 보고)
  - 이미지·경로는 상대참조 `./` 사용. 용어·출처는 실제 확인된 것만 기재(미확인 시 공란)

## [출력]

- **레포트**: `reports/{YYMMDD}/{관리그룹}/cost-analysis.md`
  - `{YYMMDD}` = **레포트 생성일(오늘)** 6자리. `{관리그룹}` = PDF에서 자동 추출한 관리그룹명
  - 경로 변수(`{YYMMDD}`·`{관리그룹}`) 미확정 시 저장 금지. 관리그룹명의 특수문자(`/ \ : * ? " < > |`)는  
    `-`로 치환해 디렉토리 안전화
- **관점별 이미지**: 위와 동일 폴더에 `{번호}.{name}.png` (레포트가 `./`로 참조)

## [제약조건]

- MUST: 
  - 레포트에 **종합분석 결과·최적화 추천·관점별 분석 결과** 3부를 모두 포함
  - 관점별 블록은 TL;DR + 이미지 + 아젠다 6섹션(3·4·5 필수)을 빠짐없이 포함
  - PDF에서 읽은 수치만 인용, 관리그룹 PDF 자동추출, 생성일 기준 경로, 관점 수 가변 처리
  - 팀원 배정 워크플로우를 실제 Agent 호출로 실행(S1 병렬 / S2~S4 순차 경계 준수)
  - 추가정보나 내 의사결정이 필요하면 반드시 요청
- MUST NOT: 
  - 원본에 없는 수치 창작, 아젠다 섹션 순서 변경, PDF에 없는 관점 임의 생성, 변수 미확정 저장
- 완료조건: 출력 경로에 `cost-analysis.md`와 관점별 PNG가 실제 생성되고,  
  3부 구성 + 관점별 필수 3섹션 존재를 Read/grep으로 확인함

## [예시]

**관점별 블록 예시 (Summary 페이지)**

```
### 1. Summary — 요약

한 줄 요약(TL;DR): 총 실질비용 81.1천·절감 1.08천, FTK Fabric이 최대 비용원, 할인 활용은 아직 부족함.

![](./01.summary.png)

#### 1. 개요
- 이번 기간 총 실질비용·절감을 한눈에 보는 표지 대시보드임.
- 데이터 범위: 2024-06-01 ~ 2024-07-08 / 필터 All / USD

#### 3. 분석 요약
- 실질비용 81.1천, 총절감 1.08천(절감률 ~1.3%). FTK Fabric 구독 합계 41,933(전체 절반 이상).

#### 4. 시사점
- 절감률 1.3%로 할인 활용 미미 → 정가 비중 과다. 대부분 구독 절감 0의 할인 사각지대.

#### 5. 권고사항
- FTK Fabric부터 드릴다운(→Charge breakdown), 약정 커버리지 권장값 확인 → 구매 실행은 Optimize 이관.
```

**최적화 추천 표 예시**

| 우선순위 | 대상 | 조치 | 예상효과 | 단계 |
|---|---|---|---|---|
| 1 | Microsoft Fabric | 용량(SKU) 재검토 | 최대 비용원(52%) 절감 | Optimize 이관 |
| 2 | 미약정 Compute | RI/SP 커버리지 확대(권장 70%부터) | 약정 절감 확대 | Optimize 이관 |
