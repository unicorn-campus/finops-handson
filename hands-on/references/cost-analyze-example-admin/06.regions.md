# 06. Regions — 지역별 비용 분포

> 페이지: Regions · 데이터 범위: 2024-06-01 ~ 2024-07-08(앞 페이지 공통) / 앞 페이지와 동일 필터 / 통화 원본 미표기(천 단위 표시)  
> 원본: CostManagementConnector.pbix (FinOps Toolkit) · Inform 단계 비용 가시화  
> 📌 한 줄 요약(TL;DR): 미국 리전(East US 중심)에 비용이 집중되고 6/23 급등이 재확인되며, 지도는 테넌트 설정으로 비활성인 화면임.

![](./06.regions.png)

## 1. 개요
- 목적: "어느 리전(Azure 데이터센터 지역)에서 비용이 발생하는가"를 보는 화면임.  
  리전은 가격 · 규정 준수 · 지연 · 전송(egress) 비용에 직접 영향을 줌.
- 데이터 범위: 청구기간 2024-06-01 ~ 2024-07-08(원본 10번 페이지 기준 앞 페이지 공통) / 앞 페이지와 동일 필터 /  
  통화 단위는 원본 미표기(값은 천 단위 표시).

## 2. 화면 구조·차트 읽는 법
- 상단: 동일 필터 + 핵심 지표 Effective cost 81.1천 / Total savings 1.08천.
- 좌측 가운데: 일자별 누적 막대 차트 — 색상이 리전을 구분함(6/23 급등 확인 가능).
- 좌측 하단: RegionName별 6월/7월/합계 표.
- 우측: Azure Maps 지도 시각화(현재 비어 있음).
- 읽는 법: 막대의 색 = 리전, 높이 = 해당 일자 비용. 표의 행 = 리전, 합계 열 = 리전 누적 비용.
- 지도 공백 사유: "Azure Maps 시각적 개체를 사용하려면 관리자에게 문의하여 테넌트 스위치를 켜세요" 오류 —  
  테넌트 관리자 설정으로 Azure Maps가 꺼져 지도가 렌더링되지 않음. 지도는 부가 시각화이며  
  비용 데이터는 좌측 표·차트로 정상 확인 가능함.

## 3. 분석 요약
> What · 데이터가 보여준 사실(해석 배제)

- 핵심 지표: Effective cost 81.1천, Total savings 1.08천, 리전 합계 81,088.9(스크롤 아래 리전 더 존재).
- 리전별 합계(상위 일부):

| 리전 | 합계 |
|---|---|
| East US | 5,266.3 (US 최대) |
| East US 2 | 2,251.4 |
| Central US | 1,187.4 |
| Canada East | 1,159.3 |
| Canada Central | 416.9 |
| France Central | 227.7 |
| East Asia | 207.6 |
| Brazil South | 0.1 |
| East US 2 EUAP | 0.0 |
| 합계 | 81,088.9 |

- 일자별 누적 막대에서 6/23 급등이 관찰됨(앞 페이지들과 동일 사건, 리전 관점 재확인).
- 우측 Azure Maps 지도는 테넌트 스위치 미허용으로 표시되지 않음.

## 4. 시사점
> So what · 사실의 의미·비용 리스크

- 미국 리전 집중: East US + East US 2 + Central US가 상위 → 05번 리소스 표의 "East US" 다수와 일치.  
  특정 리전 장애·가격 변동 시 비용 리스크가 집중됨.
- 6/23 급등: 특정 리전에서 워크로드가 확장된 신호. 앞 페이지들과 같은 사건이 리전 관점에서도 재확인됨.
- 리전 = 가격 차이: 같은 서비스도 리전별 단가가 다름 → 저렴한 리전으로 이전 시 절감 기회.
- 규정 준수·데이터 주권: 특정 데이터는 특정 리전에만 둬야 하는 규제 존재 → 리전 분포가 준수 여부에 영향.
- 전송(egress) 비용: 리전 간 데이터 이동 시 대역폭 과금 → 리전 분산이 심하면 비용 리스크.

## 5. 권고사항
> Now what · Inform 단계 실행 행동(실행은 Optimize 이관 명시)

- (우선순위 1) 리전 집중도 점검: East US 계열 상위 리전의 워크로드가 규정·가용성 요건에 맞는지 확인함.
- (우선순위 2) 6/23 급등 원인 리전 식별: 어느 리전에서 워크로드가 확장됐는지 일자별 막대로 추적함.
- (우선순위 3) 전송비 확인: 리전 분산이 크면 07번 Charge breakdown에서 네트워크/egress 비중을 확인함.
- (운영) 지도 활성화 필요 시: Entra 관리자가 "Azure Maps 데이터 전송" 테넌트 스위치를 허용해야 함  
  (데이터가 테넌트 지리 경계 밖에서 처리될 수 있다는 고지 포함).
- Inform → Optimize 이관 포인트: "저렴 리전 이전 검토"는 지연·규정을 함께 평가한 뒤  
  Optimize 단계의 실제 이전 실행으로 넘김.

## 6. 용어·출처
- 리전(Region): Azure 데이터센터가 위치한 지리적 지역. 서비스 단가·규정·지연에 영향.
- egress(전송 비용): 리전/영역 밖으로 데이터가 나갈 때 부과되는 대역폭 과금.
- 데이터 주권: 데이터를 특정 국가·지역 경계 안에 보관·처리해야 하는 규제 요건.
- Azure Maps 테넌트 스위치: Power BI Azure Maps 시각화 사용을 테넌트 단위로 켜고 끄는 관리자 설정.
- 출처(공식 문서):
  - Azure Maps Power BI 시각적 개체: https://learn.microsoft.com/azure/azure-maps/power-bi-visual-get-started
  - Azure 대역폭(데이터 전송) 요금: https://azure.microsoft.com/pricing/details/bandwidth/
  - FinOps Toolkit 개요: https://learn.microsoft.com/cloud-computing/finops/toolkit/finops-toolkit-overview
