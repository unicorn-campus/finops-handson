# 14. Shared reservation recommendations for VMs — 공유 예약 권장(3년·전 구독 공유)

> 페이지: Shared reservation recommendations for virtual machines · 데이터 범위: Lookback Last30Days · 필터 Term=3 year·Scope=Shared · 통화 원본 미표기
> 원본: CostManagementConnector.pbix (FinOps Toolkit) · Inform 단계 비용 가시화
> 📌 한 줄 요약(TL;DR): 3년 약정 시 5.82천 절감 가능(현재의 5배)이며, 최대 기회는 NC24rs_v3 GPU VM(=leapv2)이고 권장값의 70%부터 단계 구매함.

![](./14.reservation-coverage(shared).png)

## 1. 개요
- 이 페이지의 목적: 11~13번이 "가진 약정 관리"였다면, 이건 "예약을 더 사면 얼마 아끼나?"를 알려주는
  권장(추천) 엔진임. 앞서 다룬 "적정 약정 수준" 논의의 실전 화면임.
- 데이터 범위: Lookback Last30Days(최근 30일 사용량 기반) / 필터 Term=3 year·Scope=Shared / 통화 원본 미표기.

## 2. 화면 구조·차트 읽는 법
- 상단(조건+지표): Term 3 year / Lookback Last30Days와 Potential savings / Contracted cost 지표.
- 차트 ① Location > Flexibility group > Size(VM 계열별 커버리지): 계열별
  **Committed Usage(옅은 파랑, 이미 약정)** vs **Other(짙은 남색, 미약정)**를 비교함.
- 차트 ② CPU hours(시간별 커버리지 추이): Committed(옅은) vs Other(짙은)의 시간대별 추이.
- 하단(표): 권장 목록(최근 30일 기반) — 열은 Flexibility Group / Location / SKU / Rec Qty / Contracted / Potential savings임.
- "Shared" 의미: 예약을 **모든 구독이 공유**하도록 구매하는 범위임(특정 구독 전용은 15번). 공유는 유연성이 높음.

## 4. 시사점
> So what · 사실의 의미·비용 리스크

- **잠재 절감 5.82천은 현재 실제 절감(1.07천)의 5배 이상**임 → 약정 확대 기회가 매우 큼(12번 결론과 일치).
- **최대 기회 = NC24rs_v3(GPU VM)**임 — 3년 예약 시 4,367 절감. 08번 AHB의 leapv2와 동일 SKU →
  이 GPU VM은 **AHB + 예약 이중 최적화** 대상임.
- **미약정(Other) 비중이 큼** → 12번의 "98.5% 미적용"을 VM 계열별로 재확인함.
- **6/23 급증분이 전부 미약정(정가)**으로 흘러감 → 늘어난 사용량이 약정 커버 밖에서 정가 부담을 키움.
- **3년 vs 1년**: 3년이 할인폭 최대(문서상 RI 최대 72%)이나 3년 고정 리스크를 감안해야 함.

## 5. 권고사항
> Now what · Inform 단계 실행 행동(실행은 Optimize 이관 명시)

- **[우선순위 1] 최대 기회 NC24rs_v3(=leapv2)를 AHB + 3년 예약 이중 최적화 후보로 지정**함(08번과 연계).
- **[우선순위 2] 전체 권장 24건(5.82천)을 전량 구매하지 않음** → 아래 단계 구매 원칙을 적용함.
- **[우선순위 3] 안정적 계열부터 약정**함 — 미약정(Other) 비중 큰 계열 중 사용 변동이 작은 것 우선.
- **Inform → Optimize 이관 포인트**: 권장 목록·잠재 절감(5.82천)과 단계 구매 계획을 Optimize 단계에 넘겨
  실제 예약 구매·재계산 루프를 실행함.

### 보충 — 적용 원칙(단계 구매 공식)
```
1. 권장값 확인 (이 표)
2. ~70%만 우선 구매 (전량 구매 금지)
3. 3일 대기 → 권장 재계산 반영
4. 부족하면 반복
5. RI와 SP 동시 구매 시 7일 간격
```

## 6. 용어·출처

### 용어
- **Reservation recommendation(예약 권장)**: 최근 사용량 기반으로 얼마를 예약하면 얼마 아끼는지 추천하는 엔진.
- **Term(기간)**: 약정 기간(1년/3년). 길수록 할인폭↑, 고정 리스크↑.
- **Scope(범위)·Shared(공유)**: 예약을 전 구독이 공유하는 범위. 여러 구독 리소스가 자동 혜택 → 유연성↑.
- **Lookback(Last30Days)**: 권장 산정 기준 기간(최근 30일 사용량).
- **Flexibility Group / Rec Qty (Normalized)**: VM 계열 묶음 / 권장 수량(정규화는 크기 유연성 반영 수량).
- **AHB(Azure Hybrid Benefit)**: 08번의 라이선스 혜택. 동일 SKU(leapv2)에서 예약과 이중 최적화 가능.
