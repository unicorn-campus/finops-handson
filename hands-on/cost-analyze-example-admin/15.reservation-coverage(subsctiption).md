# 15. Reservation recommendations (Subscription scope / 1 year) — 예약 권장(구독 범위·1년)

> 페이지: (템플릿 제목은 "Shared"로 표시) 예약 권장 · 데이터 범위: Lookback 최근 30일 · 필터 Term=1 year·Scope=Subscription · 통화 원본 미표기
> 원본: CostManagementConnector.pbix (FinOps Toolkit) · Inform 단계 비용 가시화
> 📌 한 줄 요약(TL;DR): 1년·구독 범위 권장(3.52천)을 14번 3년·공유(5.82천)와 비교해, 기간·범위 선택이 절감폭을 좌우함을 학습함.

![](./15.reservation-coverage(subsctiption).png)

> ※ 표기 주의: 파일명 `subsctiption`은 `subscription`의 오타임. 원본·PNG와 링크 일치를 위해 파일명은 유지하며, 정식 표기는 `subscription`임.

## 1. 개요
- 이 페이지의 목적: 14번과 거의 같은 화면이지만 **범위(Scope)와 기간(Term)이 달라 비교 학습용**임.
  제목은 "Shared"로 표시되나 필터에 Subscription이 있고 Term=1 year로 14번과 구분됨(템플릿 제목 미갱신).
- 데이터 범위: Lookback 최근 30일 / 필터 Term=1 year·Scope=Subscription / 통화 원본 미표기.

## 2. 화면 구조·차트 읽는 법
- 좌(차트): Location > Flexibility group > Size — 계열별 Committed vs Other 커버리지(14번과 동일 구조).
- 우(차트): CPU hours 추이 — 6/23 급증 지점도 14번과 동일함.
- 하(표): 최근 30일 기반 권장 목록 — 열은 Flexibility group / Location / SKU / Rec Qty / Contracted / Potential savings임.
- 읽는 법 핵심: 같은 사용량 데이터라도 **Term·Scope 필터가 바뀌면 권장 조합과 절감액이 달라짐**을 대조해 읽음.

## 3. 분석 요약

## 4. 시사점
> So what · 사실의 의미·비용 리스크

- **같은 사용량인데 절감이 5.82천 → 3.52천으로 감소**함 → 기간·범위 선택이 절감폭을 직접 좌우함.
- **기간(Term)**: 3년 = 할인폭 최대(절감↑)이나 3년 고정 리스크, 1년 = 절감 적지만 사업 변동 대응에 유연.
- **범위(Scope)**: 공유(Shared) = 여러 구독 자동 혜택 → 활용률↑·유연, 단일 구독 = 그 구독만 혜택 →
  팀별 귀속 명확하나 활용률 저하 위험.
- **NC24rs_v3(GPU) 미포함**은 범위·기간이 좁아지면 고가 GPU 예약이 권장에서 빠질 수 있음을 시사함.

## 5. 권고사항
> Now what · Inform 단계 실행 행동(실행은 Optimize 이관 명시)

- **[우선순위 1] 워크로드 성격별로 Term·Scope 조합을 선택**함 — 아래 선택 가이드를 기준으로 결정함.
- **[우선순위 2] 안정적·장기 워크로드는 3년+공유(14번)** 시나리오로, **변동·불확실 워크로드는 1년+공유**로 우선 검토함.
- **[우선순위 3] 특정 팀 전용 자원은 구독 범위**로 귀속을 명확히 하되 활용률 저하를 모니터링함.
- **Inform → Optimize 이관 포인트**: 14·15번 시나리오 비교 결과(절감폭·유연성 트레이드오프)를 Optimize 단계에
  넘겨, 실제 구매 조합(기간·범위)을 확정하고 14번의 단계 구매 원칙(70%·3일 대기)을 적용함.

## 6. 용어·출처

### 용어
- **Term(기간)**: 약정 기간(1년/3년). 3년 할인폭 최대·고정 리스크, 1년 유연·절감 적음.
- **Scope(범위)**: 예약 적용 범위. Shared(전 구독 공유) vs Subscription(단일 구독).
- **Potential savings(잠재 절감)**: 권장 예약을 다 사면 아낄 수 있는 금액.
- **Contracted cost**: 권장대로 약정할 때 부담해야 할 계약 비용.
- **Flexibility group / Rec Qty**: VM 계열 묶음 / 권장 수량.

### 보충 — 2대 의사결정 축 선택 가이드
```
안정적·장기 워크로드   → 3년 + 공유  (절감 최대)
변동·불확실 워크로드   → 1년 + 공유  (유연 + 활용률)
특정 팀 전용 자원      → 구독 범위    (귀속 명확)
```
