## 상단 조건 + 지표
- **Term: 3 year** / **Lookback: Last30Days** (3년 약정, 최근 30일 사용량 기반 권장)
- **Potential savings: 5.82천** — 권장 예약을 다 사면 아낄 수 있는 금액
- **Contracted cost: 9.77천** — 그때 약정해야 할 금액

> **주목**: 잠재 절감 **5.82천**은 현재 실제 절감(1.07천)의 5배 이상 → 약정 확대 기회가 매우 큼 (12번 결론과 일치)

## "Shared" 의미
- **공유(Shared) 범위** = 예약을 모든 구독이 공유하도록 구매 (특정 구독 전용은 15번)
- 공유가 유연성↑ (여러 구독 리소스가 자동으로 혜택 받음)

## 2개 차트

### ① Location > Flexibility group > Size (VM 계열별 커버리지)
- 계열별 **Committed Usage(옅은 파랑, 이미 약정)** vs **Other(짙은 남색, 미약정)**
- DSv3 Series: 약정 7.2천 + 미약정 12.7천 → 미약정이 더 큼
- BS Series High Memory·NCSv3 등은 거의 전부 미약정(Other) → 약정 여지 큼

### ② CPU hours (시간별 커버리지 추이)
- Committed(옅은) ~0.4천 vs Other(짙은) 1.4~2.0천
- **6/23 급증**(Other 1.4→2.0천) = 지금까지 추적한 그 사건. 늘어난 사용량이 전부 미약정(정가)

## 인사이트 (읽는 법)
1. **최대 기회 = NC24rs_v3 (GPU VM)** — 3년 예약 시 4,367 절감. 08번 AHB의 leapv2와 동일 SKU → 이 GPU VM은 AHB+예약 이중 최적화 대상
2. **전체 권장 24건 사면 5.82천 절감** — 단, 앞서 공식 권고대로 권장값의 ~70%만 우선 구매 → 3일 대기 → 반복
3. **Rec Qty Normalized(78)** — 정규화 크기 기준 수량 (인스턴스 크기 유연성 반영)
4. **미약정(Other) 비중 큼** — 12번의 "98.5% 미적용"을 계열별로 재확인. 안정적 계열부터 약정
5. **3년 vs 1년** — 3년이 할인폭 최대(문서상 RI 최대 72%). 단 3년 고정 리스크 감안

## 적용 원칙 (앞서 공식 근거)
```
1. 권장값 확인 (이 표)
2. ~70%만 우선 구매 (전량 구매 금지)
3. 3일 대기 → 권장 재계산 반영
4. 부족하면 반복
5. RI와 SP 동시 구매 시 7일 간격
```
