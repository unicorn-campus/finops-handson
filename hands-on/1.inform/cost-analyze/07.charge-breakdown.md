# 07. Charge breakdown (요금 분해)

CostManagementConnector 샘플 보고서의 **Charge breakdown** 페이지 해설임.
가장 정보 밀도가 높은 화면 — 비용을 **4가지 비용 지표**로 나누고, **Sankey(흐름) 다이어그램**으로 최종 원인까지 분해함.

## 상단 요약 — 4가지 비용 지표 (FinOps 핵심 개념)

**Cost summary (비용 요약)**

| 지표 | 값 | 의미 |
|---|---|---|
| **Billed cost (청구 비용)** | **79.86천** | 실제 청구서 금액 (amortization·시점 반영) |
| **List** | 82.16천 | 정가(PAYG 공개가) — 할인 전 |
| **ContractedCost** | 82.16천 | 협상 단가 적용 후 |
| **EffectiveCost** | 81.09천 | 약정 할인까지 적용한 실질 비용 (분석·showback 기준) |

**Savings summary (절감 요약)**

| 지표 | 값 | 의미 |
|---|---|---|
| **Total savings** | 1.08천 | 총 절감 |
| **Negotiated discounts** | **0.00천** | 협상 할인 = 없음 (List=Contracted) |
| **Commitment discounts** | 1.07천 | 절감 전부가 약정에서 발생 |

> **비용 지표 흐름**: `List(정가) → Contracted(−협상) → Effective(−약정) → Billed(청구서)`
> 여기선 협상 할인 0 → List=Contracted이고, 약정으로만 1.07천 절감 → Effective 81.09천

## 가운데 — Sankey 분해 트리 (좌→우 드릴다운)

`EffectiveCost`가 어떻게 쪼개지는지 흐름으로 표시. 상단 브레드크럼(ChargeCategory→PricingCategory→…→ServiceName)으로 단계별 필터됨.

```
EffectiveCost 81,088.9
├ [ChargeCategory] Usage 81,090.8 · Purchase 0.0 · Adjustment -1.9
├ [PricingCategory] Standard 79,824.8 · Committed 1,256.9 · Dynamic 9.1
├ [CommitmentDiscount] (공백) 79,824.8   ← 약정 미적용분
└ [ServiceCategory] Analytics 43,720 · Compute 17,108 · Databases 9,742 · Web 4,098 · Integration 2,173
      └ [ServiceName] Microsoft Fabric 42,296.8 · Azure Synapse Analytics 1,423.6
```

### 여기서 드디어 밝혀진 최종 범인
- **Microsoft Fabric = 42,296.8** → 지금까지 "FTK Fabric 구독 / Analytics 대분류"로 추적하던 최대 비용의 실체가 Microsoft Fabric 서비스임이 확정됨
- 01→02→03→07로 오며 총액 → 구독 → 서비스대분류 → 서비스명까지 좁혀진 것

## 새 용어 (분류 축)

| 축 | 값 | 뜻 |
|---|---|---|
| **ChargeCategory** | Usage / Purchase / Adjustment | 사용료 / 구매 / 조정(환불 등) |
| **PricingCategory** | Standard / Committed / Dynamic | 정가 / 약정적용 / 스팟·변동 |
| **CommitmentDiscount** | (공백) 등 | 어떤 약정이 적용됐나 (공백=미적용) |

## 인사이트 (읽는 법)
1. **절감의 100%가 약정** — 협상 할인 0 → 볼륨 협상 여지 존재 (계정팀과 협상 시 추가 절감 가능)
2. **약정 적용률 낮음** — Committed 1,256.9 vs Standard 79,824.8 → 대부분 정가(Standard) → RI/SP 커버리지 확대 여지 큼
3. **Microsoft Fabric 단일 42.3천** — 전체의 52%. Fabric 용량(SKU) 최적화가 최대 절감 레버
4. **Billed(79.86천) ≠ Effective(81.09천)** — amortization·시점 차이. 회계는 Billed, 최적화 분석은 Effective 사용

**한 줄 요약**: Charge breakdown은 "4대 비용 지표(List/Contracted/Effective/Billed) + Sankey 분해로
최종 범인 = Microsoft Fabric(42.3천) 확정, 절감은 전부 약정·협상할인 0·약정 적용률 낮음"을 보여주는 심층 분해 화면.
