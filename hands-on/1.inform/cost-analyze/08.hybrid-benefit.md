# 08. Azure Hybrid Benefit for Windows Server (AHB)

CostManagementConnector 샘플 보고서의 **Azure Hybrid Benefit for Windows Server** 페이지 해설임.
AHB는 라이선스 비용 최적화 기법 — 보유 중인 온프레미스 Windows Server 라이선스(SA 포함)를 Azure VM에
재활용해 OS 라이선스 비용을 절감함 (약정/RI와는 별개 축).

![](08.hybrid-benefit.png)

---

## 개념 먼저 — AHB란
- 이미 산 **Windows Server(또는 SQL Server) 라이선스**를 Azure VM에 적용 → Azure가 부과하는 OS 라이선스 요금 면제
- Windows VM에서 최대 ~40% 이상 절감 가능
- 단, 라이선스는 유한하고 **1 VM당 최소 8코어(vCPU) 단위**로 소모됨 → 어디에 쓰느냐가 중요

## 상단 지표
- **Enabled (last day): 6** — AHB 켜진 리소스 수
- **vCPUs used (last day): 16.00** — AHB로 커버 중인 vCPU

## 3개 차트 — 낭비 vs 기회

### ① SKUs enabled with <8 vCPU (낭비 경고)
- AHB를 켰지만 vCPU가 8 미만인 VM들 ((공백)=1개(4vCPU 3개), Standard_...=2개(2vCPU))
- 문제: 라이선스는 VM당 최소 8코어를 잡아먹는데, 4·2 vCPU 작은 VM에 쓰면 라이선스 낭비
- → 작은 VM에서 빼고 큰 VM으로 옮기는 게 효율적

### ② Not enabled with 8+ vCPUs (놓친 기회)
- vCPU 8 이상인데 AHB가 꺼진 VM — Standard_...=1개(24 vCPU)
- → 여기에 AHB를 켜면 즉시 절감 = 최우선 조치 대상

### ③ AHB vCPUs consumed (최근 30일)
- 라이선스로 커버한 vCPU 추이. 6/23 부근 급감 후 회복 등 변동 관찰

## 인사이트 (읽는 법 / 조치)
1. **최우선 조치 — leapv2에 AHB 켜기**: 24 vCPU 대형 VM인데 AHB 미적용, 비용 390.6으로 표에서 가장 큼 → 켜면 라이선스 절감 즉시 발생
2. **작은 VM 재검토**: 4·2 vCPU VM에 AHB가 켜져 있음 → 8코어 최소 소모 규칙상 비효율. 라이선스를 큰 VM으로 재배치 검토
3. **라이선스는 유한 자원** — "8+ vCPU VM에 우선 배치"가 라이선스당 절감 극대화 원칙
4. **약정과 별개 축** — 07번의 약정(RI/SP)과 함께 AHB는 라이선스 절감으로 추가 레버. 둘 다 활용해야 총절감 극대화

## 절감 레버 정리 (지금까지)
```
① 약정 할인(RI/SP)   ← 07번: 적용률 낮음, 확대 여지
② 협상 할인          ← 07번: 0, 협상 여지
③ AHB(라이선스)      ← 08번: leapv2 미적용 = 즉시 기회
```

**한 줄 요약**: AHB 페이지는 "leapv2(24vCPU) 미적용 = 즉시 절감 기회, 작은 VM들은 라이선스 낭비,
8+ vCPU에 우선 배치"를 보여주는 라이선스 최적화 화면.
