# 약정 구매 절차 — RI/SP 구매 시뮬레이션 & Amortized 분배 개념

> 계약 유형: **EA(Enterprise Agreement)** 기준 · 대상 제품: **VM 예약 중심**(SP는 Azure 절감 플랜 for compute)
> 실습 방식: **포털 탐색·캡처 병행** · ⛔ **결제 미수행(시뮬레이션)** — 어느 절차에서도 최종 **`구매(Buy)` 버튼은 클릭하지 않음**
> 단계: FinOps **Optimize(최적화)** — Rate Optimization(요율 최적화) 역량
>
> ⚠️ **최신성(2026-07-19 기준)**: **2026-07-01부** 일부 VM 시리즈의 예약(RI) **신규 구매·갱신이 중단**됨.
> 기존 예약은 기간 만료까지 계속 적용되나 해당 시리즈는 **구매 목록에 미표시**될 수 있음 → 실습 대상 SKU 선택 시 유의.
> ✅ 본 문서의 한국어 UI 라벨은 **2026-07-19 실제 Azure 포털(EA·한국어)** 구매 화면으로 검증함(구매 클릭 없음).

## 0. 학습 목표 (WHY)

- **왜 약정인가**: 온디맨드(정가) 대비 선지불·기간 약정을 조건으로 **요율 할인**을 받아 안정 워크로드 비용을 낮춤.
  약정은 **낭비를 없애는 도구가 아니라 요율을 낮추는 도구**임(먼저 Right-sizing 후 약정).
- 이 실습으로 얻는 것
  1. RI(예약)와 SP(절감 플랜)의 **차이와 선택 기준**을 구분함.
  2. 구매 3대 축 — **기간(Term) · 범위(Scope) · 결제옵션(대금 청구 주기)** — 을 실제 포털 화면에서 선택함.
  3. 구매를 실행하지 않고도 **Amortized(상각) 분배**로 비용이 어떻게 인식되는지 이해함.
- **완료 기준**: RI·SP 각각 **구매 직전 "검토 + 구매" 화면**까지 도달(구매 클릭 금지)하고,
  Actual vs Amortized 관점 차이를 설명할 수 있음.

## 1. 사전 개념 — RI vs SP 무엇을 언제

| 구분 | RI (Reservation, 예약) | SP (Savings Plan, 절감 플랜) |
|---|---|---|
| 약정 대상 | **특정 리소스**(VM SKU·리전 등) | **시간당 컴퓨트 지출액**(compute) |
| 유연성 | 낮음(인스턴스 종류·계열·리전 고정 성향) | 높음(계열·컴퓨트 서비스·리전 이동에도 자동 적용) |
| 할인폭 | **완전 활용 시 최대**(동일 대상에서 예약 할인폭이 더 큼) | 예약보다 유연한 대신 상대적으로 낮음 |
| 적합 워크로드 | **안정·연속** 실행, 변경 예상 없음 | **동적·변동**, 다양한 계열/서비스, 리전 이동 |
| 적용 방식 | 매칭되는 리소스에 예약 할인 적용 | 시간당 약정액까지 컴퓨트 사용분에 할인율 적용 |
| 취소·교환 | **제한적** 취소·교환·환불 가능 | **취소·교환 불가**(예약 → SP 트레이드인만 가능) |

- **선택 기준(공식 문서)**
  - 인스턴스 종류·계열·리전 변경이 없고 계속 돌아가는 워크로드 → **RI**(완전 활용 시 최대 절감).
  - 워크로드가 동적·진화하거나 여러 계열/컴퓨트 서비스를 쓰거나 리전을 옮김 → **SP**(유연·자동 최적화).
- **적용 순서 원칙(중요)**: ① **Right-sizing으로 낭비·과대 리소스 제거** → ② 저활용 예약을 SP로 **교환(trade-in)**
  → ③ 정리된 기준선에 맞춰 **신규 약정 구매**(변동 워크로드는 SP, 안정·연속 워크로드는 신규 RI도 선택지).
  낭비를 먼저 없앤 뒤 약정해야 "이론상 할인"이 아닌 "실제 절감"이 됨.

## 2. 구매 3대 의사결정 축

구매 화면의 핵심 선택은 아래 3가지임. 시뮬레이션에서 각 값을 바꿔 보며 예상 비용·절감 변화를 관찰함.

### 2-1. 기간 (Term)

- 선택지: **1년 / 3년**(일반 VM 기준. **HBv2 시리즈 한정**으로 5년 옵션도 존재).
- 트레이드오프: 3년은 **할인폭 최대**이나 **3년 고정 리스크**(사업 변동 대응성↓).
  1년은 **유연**하나 절감폭이 상대적으로 작음.
- 판단: 안정·장기 워크로드 → 3년, 변동·불확실 워크로드 → 1년.

### 2-2. 범위 (Scope) — 공유 vs 단일

예약 할인이 **어디에 적용되는지**를 정함. EA 기준으로 4가지 범위가 있음.
(포털 라벨: 예약은 **`범위`**, 절감 플랜은 **`적격 리소스에 적용`**)

| 범위(Scope) | 적용 대상 | 특징 |
|---|---|---|
| **공유(Shared)** | 청구 컨텍스트 내 **모든 적격 구독** | 여러 구독 리소스가 자동 혜택 → **활용률↑·유연**. EA는 청구 컨텍스트 = **Enrollment(등록)** |
| **단일 구독(Single subscription)** | 선택한 **구독 하나** | 그 구독만 혜택. 팀·구독 단위 **귀속 명확**, 활용률 저하 위험 |
| **단일 리소스 그룹(Single resource group)** | 선택한 **리소스 그룹 하나** | 가장 좁은 범위. 특정 프로젝트 전용 |
| **관리 그룹(Management group)** | 관리 그룹 계층 + 청구 범위에 속한 구독들 | 조직 계층 단위 적용(읽기 권한 + 청구 구독의 예약 소유자/구매자 권한 필요) |

- **함정**: 범위를 좁게(단일 구독/RG) 잡으면 그 범위에 매칭되는 리소스가 없거나 줄어들 때 **활용률↓ = 낭비**가 됨.
  공유는 유연하지만 **팀별 비용 귀속(차지백)**이 흐려지므로, 귀속이 중요하면 상각(Amortized) 데이터로 소비 주체에 되돌림(5장).
- **참고**: 범위는 **구매 후에도 변경 가능**하며, 범위 재조정은 **상거래(과금) 행위가 아님**(기간·조건 불변).
- ⚠️ **예약 ≠ 용량 보장**: 예약(RI)은 **요율(청구) 할인일 뿐 컴퓨트 용량을 예약·보장하지 않음**.
  리전/가용성 영역 용량이 없으면 예약을 샀어도 배포가 실패할 수 있음 →
  용량 보장이 필요하면 **온디맨드 용량 예약(On-demand capacity reservation)** 을 별도 사용(A-5의 '용량 우선순위'와 다른 개념).

### 2-3. 결제 옵션 (대금 청구 주기 / Billing frequency) — 선불 vs 월별

- 포털 라벨: **`대금 청구 주기`**. 선택지: **월별(Monthly)** 또는 **전액 선불(All upfront)**.
- **총액은 동일함** — 월별은 총 약정액을 **1년=12회 / 3년=36회**로 균등 분할해 납부할 뿐, 할인 총액 차이 없음.
- 선택 관점
  - 전액 선불: 현금 흐름 여유가 있고 회계상 선지출을 감수 → 초기 일시 계상.
  - 월별: 현금 흐름 평탄화(권장 기본값이 되는 경우 많음). MCA는 환율에 따라 월 납부액이 달라질 수 있음(EA는 해당 표기 상이).
- ⚠️ **이 실습에서는 어떤 옵션을 골라도 실제 납부는 발생하지 않음** — 구매 클릭 전까지는 견적일 뿐임.

## 3. 실습 A — RI(예약 VM 인스턴스) 구매 시뮬레이션

> 목표: VM 예약을 **구매 직전 검토 화면**까지 구성. ⛔ **`구매` 클릭 금지**.

> **사전 조건(EA)**: EA 구독은 EA 포털에서 **"Add Reserved Instances(예약 구매 허용)" 설정이 활성화**돼야 함.
> 비활성 시 해당 구독의 **EA 관리자**여야 구매 가능. 또한 청구 구독에 **예약 구매자/소유자** 권한 필요.
> 💡 **실환경 확인(2026-07-19)**: 구매 권한(`Microsoft.Capacity/register/action`)이 없으면 제품 목록에
> **"예약 구매 사용 안 함"** 이 표시되고 **`카트에 추가`가 비활성**됨 → EA 관리자에게 예약 구매 허용·역할 부여를 요청.
> (2026-07-01부 일부 VM 시리즈는 신규 구매·갱신 중단 — 목록 미표시 가능)

### A-1. 예약 화면 진입 → 추가

- 포털 상단 검색창에 **"예약(Reservations)"** 입력, 또는 **모든 서비스(All services) > 예약** 으로 진입.
- 상단 도구모음의 **`+ 추가(Add)`** 클릭(예약이 없으면 빈 화면의 **`지금 구매`** 버튼도 동일 진입) → **"예약 구매"** 화면이 열림.

![](images/ri-01-reservations-add.png)
*예약(Reservations) 목록 화면과 `+ 추가` 버튼(도구모음: 카트 보기·Exchange·자동 갱신·역할 할당 등)*

### A-2. 제품 선택 — 가상 머신

- **"예약 구매"** 화면 상단 탭은 **`카트`** · **`검토 + 구매`**. 제품 목록(**예약**)에서 **가상 머신(Virtual machine)** 선택.
  (SQL Database·Cosmos DB·Blob Storage·Dedicated Host 등 다른 예약 제품도 같은 목록에 있음)

![](images/ri-02-select-product.png)
*"예약 구매" 제품 목록에서 "가상 머신" 선택*

### A-3. 범위·청구 구독 선택 (EA)

- 구성 화면 **"구매할 제품 선택"** 상단에 **`범위`**(기본값 **공유**) 드롭다운과 **`청구 구독`** 이 있음.
- **`청구 구독(Billing subscription)`**: 예약을 청구할 구독(예약 구매자/소유자 권한 필요).
- **`범위(Scope)`**: 2-2의 4가지 중 선택 — 실습에서는 **공유** 와 **단일 구독** 을 각각 바꿔 보며
  하단 예상 절감·적용 대상 변화를 관찰함.

![](images/ri-03-scope-billing.png)
*"구매할 제품 선택"의 범위(공유 등)·청구 구독 선택 영역*

### A-4. 제품 구성 — 지역·SKU·수량·기간

- 탭 **`권장`**(최근 사용량 기반 권장, 기준 **30일 사용량**) / **`모든 제품`**(직접 선택). 필터칩: **`지역` · `기간` · `대금 청구 주기`**.
- **지역(Region)**: 예약할 리전(예: Korea Central).
- **제품(SKU)**: VM 크기(예: 가이드 표준 `Standard_B2s` 등) 또는 **인스턴스 크기 유연성 그룹**.
- **수량(Quantity)**: 포털이 최근 사용량 기반 **권장 수량**을 제시함(**권장** 탭). **모든 제품** 탭에서 직접 지정도 가능.
- **기간(Term)**: **1년 / 3년** 선택(2-1. HBv2 한정 5년 예외).
- ⚠️ **2026-07-01부** 일부 VM 시리즈는 예약 신규 구매·갱신 중단 → **목록에 안 뜨는 시리즈**가 있을 수 있음(캡처 시 확인).

![](images/ri-04-product-config.png)
*권장/모든 제품 탭과 지역·기간·대금 청구 주기 필터, 권장 수량 표시*

### A-5. 최적화 대상 (Optimize for)

- 기본값 **인스턴스 크기 유연성(Instance size flexibility)**: 같은 VM 크기 그룹의 다른 VM에도 할인 적용 → **활용률↑**.
- **고급 설정(Advanced settings)** 에서 **용량 우선순위(Capacity priority)** 로 변경 가능 —
  데이터센터 용량을 우선 확보. **단, 용량 우선순위는 범위가 "단일 구독"일 때만 사용 가능**.
  (이 옵션은 배포 우선순위 조정일 뿐, 2-2의 '온디맨드 용량 예약'과는 별개 개념임)

![](images/ri-05-optimize-for.png)
*Optimize for: 인스턴스 크기 유연성(기본) vs 용량 우선순위(고급)*

### A-6. 결제 빈도 선택 (대금 청구 주기)

- **`대금 청구 주기`** 에서 **월별(Monthly)** 또는 **전액 선불(All upfront)** 선택(2-3). 총액 동일.

![](images/ri-06-billing-frequency.png)
*대금 청구 주기: 월별 vs 전액 선불(총 약정액 동일)*

### A-7. 카트에 추가 → 검토 화면에서 정지 ⛔

- **`카트에 추가(Add to cart)`** 클릭(카트 담기는 **과금 아님**, 되돌릴 수 있음. 담긴 수는 상단 **`카트 보기`** 에서 확인).
- 상단 탭 **`검토 + 구매(Review + buy)`** 로 이동 → 요약(범위·기간·수량·대금 청구 주기·예상액) 확인.
- 🛑 **여기서 정지.** 최종 **`구매(Buy)`** 버튼은 **클릭하지 않음**(시뮬레이션 종료 지점).

![](images/ri-07-review-stop.png)
*검토 + 구매 화면 — 이 화면까지만 도달하고 `구매`는 누르지 않음*

## 4. 실습 B — SP(Azure 절감 플랜 for compute) 구매 시뮬레이션

> 목표: 컴퓨트 절감 플랜을 **구매 직전 검토 화면**까지 구성. ⛔ **`구매` 클릭 금지**.
> (포털 한국어 표기는 **"절감 플랜"**(목록 항목은 **"절약 플랜"**)이며, 예약과 동일한 **"예약 구매"** 화면에서 진입함)

### B-1. 절감 플랜 화면 진입 → 추가

- 포털 검색창에 **"절감 플랜(Savings plans)"** 입력 → 진입 → **`+ 추가`**
  (예약과 동일한 **"예약 구매"** 화면에서 **절감 플랜** 섹션의 **`절약 플랜`** 항목 선택으로도 진입).
- SP는 **Azure 절감 플랜 for compute** 전용이라 **별도 제품 선택 없이** 구성 화면 **"플랜 추가"** 가 바로 열림.
- **`이름(Name)`**: 식별용 친숙한 이름(기본값 예: `SavingsPlan_2026-07-19_17-34` 자동 채움).

![](images/sp-01-savings-plans-add.png)
*"플랜 추가" 구성 화면 — 이름·청구 구독 입력 시작*

### B-2. 범위·청구 구독

- **`청구 구독`**(절감 플랜 구매자 권한 또는 구독 소유자) 선택.
- **`적격 리소스에 적용`**(=범위) 라디오 4종 중 선택:
  **① 구독의 청구 범위 전체에서 공유됨(공유)** / ② 구독에서만 가능(단일 구독) /
  ③ 선택한 구독의 특정 리소스 그룹 내 / ④ 선택한 구독의 관리 그룹에서
  (SP는 컴퓨트 사용분에 폭넓게 적용됨. **관리 그룹에는 권장값이 제공되지 않음**).

![](images/sp-02-scope-billing.png)
*"적격 리소스에 적용"(범위) 라디오 4종·청구 구독*

### B-3. 기간 + 시간당 약정(Hourly commitment)

- **`기간(Term)`**: **1년 / 3년**.
- **`USD 기준 시간당 약정`(Hourly commitment)**: 매시간 이 금액만큼 컴퓨트 사용을 할인율로 커버함.
  - 포털이 범위 기준 **권장값 최대 10개**를 제시함(**절약 플랜** 드롭다운). 각 권장값은
    **① 시간당 약정액 · ② 온디맨드 대비 절감률(%) · ③ 커버리지(%)** 를 함께 표시함.
  - 커버리지 %는 해당 약정 + 기존 예약/SP를 합쳐 사용량의 몇 %를 덮는지를 의미함.

![](images/sp-03-hourly-commitment.png)

## 5. Amortized(상각) 분배 개념 — 결제하지 않아도 반드시 이해

> 핵심 질문: "약정을 사면 그 비용은 청구서·비용 분석에 **어떻게** 나타나는가?"
> 같은 구매를 보는 **두 개의 렌즈**가 있음 — **Actual(실제)** vs **Amortized(상각)**.

### 5-1. 두 데이터셋 비교

| 관점 | Actual Cost (실제 비용) | Amortized Cost (상각 비용) |
|---|---|---|
| 용도 | **청구서 대사(reconcile)** — 회계 | **최적화·차지백** — 실제 소비 귀속 |
| 구매 비용 | **구매 시점에 계상**(ChargeType = `Purchase`) | **표시 안 됨**(구매 라인 = 0, 기간에 분배) |
| 할인 받은 사용분 | EffectivePrice = **0**(구매로 이미 지불) | 약정 단가로 **각 리소스에 분배** |
| 미사용분 | 별도 표시 없음 | **UnusedReservation** 유형으로 등장 |
| 선불 구매 시 모습 | 구매월에 **전액 일시** 계상 | 약정 기간(예: 3년) 전체에 **매일 균등 분배** |

### 5-2. 왜 이렇게 나뉘는가 (그림)

```
[전액 선불 RI 구매: 예 360 / 3년]

Actual(실제) 관점  ──▶  구매월에 360 통째 계상 (ChargeType=Purchase)
                        이후 그 사용료 라인은 0 (이미 지불했으므로)

Amortized(상각) 관점 ─▶  구매 라인 = 0 (표시 안 함)
                        360을 약정 기간에 균등 분배 → 매일 ≈ 0.33씩
                        실제 그 할인을 쓴 리소스/구독에 귀속
                        약정을 안 쓴 날 = UnusedReservation(낭비 가시화)
```

- **월별(Monthly) 결제**여도 개념은 동일함 — Actual은 매월 납부액이 찍히고,
  Amortized는 그 금액을 **사용 기간에 걸쳐 소비 리소스로** 분배함.

### 5-3. 비용 분석(Cost analysis)에서 전환하는 법

- 비용 분석은 **기본이 Actual cost**(청구서와 동일한 모습)임.
- **Amortized cost**로 전환하면: 구매액이 **혜택을 쓴 리소스별로 쪼개져** 보이고,
  **UnusedReservation**(미사용 약정) 유형이 새로 나타남.
- **Group by 차지 종류(ChargeType)**: 사용(Usage)·구매(Purchase)·환불(Refund) 분해.
  **Group by 예약(reservation)**: 예약분 vs 온디맨드분 분해.

### 5-4. FinOps 시사점

- **회계 마감·청구 대사** → **Actual** 을 봄(구매 스파이크는 정상적 약정 선불로 해석).
- **최적화·팀별 차지백** → **Amortized** 를 봄(중앙 구매 약정을 실제 소비 팀·리소스에 공정 배분).
- **낭비 탐지**: Amortized의 **UnusedReservation > 0** 은 "산 약정을 다 못 씀" 신호 →
  범위(scope) 조정 또는 교환/환불 검토 대상(활용률 점검과 직결).

## 6. 단계 구매 원칙 (과다 구매 방지)

권장값을 **한 번에 전량 구매하지 않음**. 실제 구매(Optimize 실행) 시 아래 **운영 권장 관행**을 따름
(아래 수치는 공식 구매 문서의 규정이 아닌 **운영 휴리스틱** — 조직 기준으로 조정).

```
1. 권장값 확인 (포털 권장 목록)
2. 권장의 ~ 70%만 우선 구매 (전량 구매 금지)
3. 3일 대기 → 사용량 반영해 권장 재계산
4. 부족하면 반복 구매
5. RI와 SP를 함께 도입할 때는 두 구매 사이 7일 간격
```

- 이유: 약정은 되돌리기 어려움(특히 **SP는 취소·교환 불가**, RI도 3년·선불일수록 제약↑).
  **과소 → 추가 구매**는 쉬우나 **과다 → 환불·교환**은 제약이 큼.

## 7. 주의·함정 체크리스트

- [ ] ⛔ **결제 미수행 확인**: RI·SP 모두 **`구매` 버튼을 누르지 않고** "검토 + 구매" 화면에서 정지했는가.
- [ ] **RI vs SP 혼동 금지**: RI=특정 리소스 고정(할인↑·유연성↓), SP=시간당 컴퓨트 지출 약정(유연성↑).
- [ ] **Actual vs Amortized 혼동 금지**: 선불은 구매월 일시 계상(Actual), 상각뷰는 기간 분배(Amortized). 비용 분석 왜곡 주의.
- [ ] **범위 신중**: 공유는 유연·활용률↑이나 귀속 흐림 / 단일은 귀속 명확이나 미매칭 시 활용률↓·낭비.
- [ ] **취소·교환 비대칭**: **SP는 취소·교환 불가**(트레이드인 수신만), **예약(RI)만 제한적 취소·교환·환불 가능**. 3년·선불일수록 신중.
- [ ] **예약 ≠ 용량 보장**: RI/SP는 요율 할인일 뿐. 용량 보장은 **온디맨드 용량 예약**으로 별도 확보(배포 실패 방지).
- [ ] **구매 권한**: 권한 없으면 **"예약 구매 사용 안 함"** + `카트에 추가` 비활성 → EA 예약 구매 허용·역할 부여 필요.
- [ ] **선(先) Right-sizing**: 약정은 낭비를 없애지 못함. 과대·유휴 리소스 정리 후 약정.

## 8. 용어·출처

### 용어
- **RI(Reservation, 예약)**: 특정 리소스(VM SKU·리전 등)를 1/3년 약정하고 요율 할인받는 선구매 상품.
- **SP(Savings Plan, 절감 플랜)**: 시간당 컴퓨트 지출액을 1/3년 약정하고 할인율을 적용받는 유연 상품.
- **Term(기간)**: 약정 기간(1년/3년, HBv2 한정 5년). 길수록 할인↑·고정 리스크↑.
- **Scope(범위)**: 할인 적용 범위(공유/단일 구독/단일 RG/관리 그룹). 포털: 예약=`범위`, SP=`적격 리소스에 적용`.
- **대금 청구 주기(Billing frequency)**: 전액 선불(All upfront) vs 월별(Monthly). 총액 동일.
- **Actual Cost(실제 비용)**: 청구서와 동일한 관점. 구매는 시점 계상(ChargeType=Purchase).
- **Amortized Cost(상각 비용)**: 구매액을 약정 기간에 분배해 소비 리소스에 귀속. UnusedReservation 유형 등장.
- **UnusedReservation**: 상각뷰에서 나타나는 미사용 약정분(낭비 신호). 100% 활용이면 수량 0.
- **온디맨드 용량 예약(On-demand capacity reservation)**: 컴퓨트 **용량 자체를 보장**하는 별도 리소스.
  예약(RI)의 요율 할인과 목적이 다르며, 함께 조합 가능.
